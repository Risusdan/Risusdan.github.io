GoogleTest：斷言列表

Posted on 2024-09-18 In GoogleTest Word count in article: 765 Reading time ≈ 3 mins.

本文列出 GoogleTest 支援的斷言。

大部分的斷言都有 EXPECT_ 和 ASSERT_ 兩種版本：

  • EXPECT_ ：在失敗時會產生非致命失敗，允許當前測試案例繼續執行
  • ASSERT_ ：在失敗時會產生致命失敗，中斷當前的測試案例

Binary Comparison

  • 也可以比較兩個指標是否相等
  • 如果用在兩個字串，會測試它們是否在相同的記憶體位置，而不是它們是否有相同的值。使用 EXPECT_STREQ 比較字串的值
  • 當比較指標是否為 NULL 時，使用 EXPECT_EQ(ptr, nullptr) 代替 EXPECT_EQ(ptr, NULL)
  • 也可以比較兩個指標是否不相等
  • 如果用在兩個字串，會測試它們是否在不同的記憶體位置，而不是它們是否有不同的值。使用 EXPECT_STRNE 比較字串的值
  • 當比較指標是否不為 NULL 時，使用 EXPECT_NE(ptr, nullptr) 代替 EXPECT_NE(ptr, NULL)

String Comparison

  • 以下斷言比較兩個 C 字串。要比較兩個 string 對象，請使用 EXPECT_EQ 或 EXPECT_NE 代替。
  • 比較字串與 NULL 時，使用 EXPECT_EQ(c_string, nullptr) 或 EXPECT_NE(c_string, nullptr)

Floating-Point Comparison

  • 由於四捨五入的誤差，兩個浮點數完全相等的機率非常低，所以 EXPECT_EQ 不適合用於浮點數比較。一般來說，為了讓浮點數比較有意義，使用者需要仔細選擇 誤差範圍。
  • GoogleTest 也提供使用 ULPs 的斷言，ULPs(Units in the Last Place) 是單位在最低有效位數的縮寫，詳細請參考 Comparing Floating Point Numbers

Exception Assertions

  • 以下斷言驗證某段程式碼是否拋出例外。需要啟用建置環境中的例外。
  • 程式碼區塊可以包含多個語句，例如：EXPECT_NO_THROW({ int n = 5; DoSomething(&n); });
