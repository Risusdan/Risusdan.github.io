Git

Posted on 2024-09-23 Edited on 2024-09-24 In VersionControl Word count in article: 1.5k Reading time ≈ 5 mins.

記錄一些常用的 Git 指令。

安裝

  • Windows: 到 Git 官網 下載安裝檔案，並依照安裝步驟進行安裝。
  • Linux: 使用套件管理工具安裝，例如在 Ubuntu 使用 sudo apt-get install git 安裝。
  • macOS: 使用 Homebrew 安裝，輸入 brew install git 安裝。

安裝完成後，開啟終端機，輸入 git --version 確認 Git 是否安裝成功。

設定 Git

設定使用者名稱與郵件地址

初次使用 Git 時，需要設定使用者名稱與郵件地址。

設定完成後，可以透過 git config --list 確認是否設定成功。

設定縮寫

使用以下指令可以設定縮寫，方便快速輸入。

未來可以透過 git co 代替 git checkout，git st 代替 git status，git 1 代替 git log --oneline --graph。

名詞解釋

  • 工作目錄(Working Directory)：實際存放檔案的那個目錄。
  • 暫存區(Staging Area)：又稱索引(index)，是 Git 用來暫存檔案的地方，我們可以透過 git add 將檔案新增到暫存區。
  • 本地儲存庫(Local Repository)：是 Git 在本地端(自己的電腦)存放檔案的地方，我們可以透過 git commit 將暫存區的檔案提交到本地儲存庫。
  • 遠端儲存庫(Remote Repository)：是 Git 在遠端(例如伺服器)存放檔案的地方，我們可以透過 git push 將本地儲存庫的檔案提交到遠端儲存庫。
  • HEAD：是一個指標，指向目前所在的提交(commit)，每次提交時，HEAD 會指向新的提交。

常用指令

初始化

用 git init 初始化目錄，讓該目錄下的檔案被 Git 管理。

查詢狀態

用 git status 查詢現在這個目錄的狀態，包含工作目錄、暫存區、遠端儲存庫的狀態。

檢視紀錄

用 git log 檢視本地儲存庫(Local Repository)的紀錄，越新的紀錄會越在上面。

用 git log --oneline 可以讓紀錄變成一行。

用 git log --oneline --graph 可以讓紀錄變成圖形。

如果只想檢視前 10 筆紀錄，可以使用 git log --oneline --n 10。

如果只想檢視某個人的紀錄，可以使用 git log --oneline --author="Your Name"。

如果想搜尋提交訊息中符合某個關鍵字的紀錄，可以使用 git log --oneline --grep="Your Keyword"。

新增檔案給 Git 管理

用 git add <file> 將檔案新增到暫存區(Staging Area)。

如果想要新增所有檔案，可以使用 git add .。

停止追蹤檔案

用 git rm --cached <file> 停止追蹤某個檔案，但不刪除該檔案。

忽略檔案的追蹤

有些檔案不想要被 Git 追蹤，例如編譯後的檔案、暫存檔案等，可以建立並編輯 .gitignore 來忽略這些檔案。

注意 .gitignore 檔案本身也需要被 Git 追蹤，所以需要將 .gitignore 檔案新增到暫存區(Staging Area)。

還要注意 .gitignore 只對尚未被追蹤的檔案有效，如果是已經被追蹤的檔案，需要先用 git rm --cached 來移除追蹤，才能生效。

提交

用 git commit -m "Commit Message" 將暫存區(Staging Area)的檔案提交到本地儲存庫(Local Repository)。

修改提交訊息

用 git commit --amend -m "New Commit Message" 修改最近一次提交的訊息。

救回被刪除的檔案

如果檔案被刪除，可以使用 git checkout <file> 來救回被刪除的檔案。 git checkout 在後面接檔名或路徑的時候，會從暫存區(Staging Area)拿出檔案，覆蓋掉工作目錄(Working Directory)的檔案。

如果想要一口氣救回所有被刪除的檔案，可以使用 git checkout .。

拆掉最近一次的提交

如果想要用「相對」的方式拆掉前一次的提交，可以使用 git reset HEAD^，後面接的 ^ 代表 HEAD 節點前一次的提交。

如果是前五次提交，可以使用 git reset HEAD~5。

如果想要用「絕對」的方式拆掉某一次的提交，可以使用 git reset <commit-id>。

reset 指令可以搭配以下三種常見參數使用：

模式 --mixed --soft --hard
從提交拆出來的檔案 丟回工作目錄 丟回到暫存區 丟掉

實際上 reset 比較像是「移動到」某個提交，那些被拆掉的檔案即便是 --hard 模式，也還是存在於本地儲存庫(Local Repository)中，只是被塞到某個角落而已(想救還是救的回來)。

如果想救回用 --hard 模式丟掉的檔案，可以透過 git reflog 找到被拆掉的 commit id，再用 git reset <commit-id> --hard 救回檔案。

檢查目前有哪些分支

用 git branch 檢查目前有哪些分支，有星星符號的表示目前所在的分支。

新增分支

用 git branch <branch-name> 新增一個分支。

重新命名分支

用 git branch -m <old-branch-name> <new-branch-name> 重新命名分支。

刪除分支

用 git branch -d <branch-name> 刪除某個分支，如果分支還沒合併到主分支，會刪除失敗。

切換分支

用 git checkout <branch-name> 切換到某個分支。

Git 在切換分支的時候主要做了兩件事：

  1. 把 HEAD 指向新的分支。
  2. 更新暫存區和工作目錄，將檔案換成目前所在分支的檔案(但切換分支之前所做的修改還是會留在工作目錄不受影響)。

也就是說，切換分支並不會影響已經在工作目錄的那些修改。

新增並切換分支

用 git checkout -b <branch-name> 新增一個分支並切換到該分支。

合併分支

用 git merge <branch-name> 將某個分支合併到「目前所在」的分支。

如果合併分支的時候希望無論如何都出現小耳朵，不要被 fast-forward 做優化，可以使用 git merge --no-ff <branch-name>。
